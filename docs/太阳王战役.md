# 太阳王战役（按关卡顺序的机制/事件/卡牌/状态总表）

> 口径：第 1、2 关以当前代码实现为准（`firstMandate` → `secondMandate`）；第 3 关为设计草案，文中已写明与代码对齐点（如 `scriptedCalendar`、`calendarYearForTurn`）。  
> 主要依据：`src/data/*`、`src/logic/*`、`src/app/gameReducer.ts`、`src/locales/*`。

## 第 1 关：旭日（`firstMandate`，1661–1675）

- 回合：15 年（每回合 1 年）
- 开局资源：财政 2 / 权力 2 / 合法性 2
- 胜利条件：财政 ≥ 6，权力 ≥ 6，合法性 ≥ 5

### 一、机制列表（按“发生顺序/优先级”）

1. **年初资源与抽牌机制**
  - 先执行年初状态效果，再把 `经费 += 财政`。
  - 抽牌次数由：权力阈值基数 + 上回合延迟修正 + 多年延迟修正 + 状态修正 +（若有）反法同盟掷骰修正组成，最少抽 1。
2. **事件填充机制（A-C 常规槽）**
  - 第 1 年固定前缀事件：`殖民贸易机遇` + `官僚拖延`。
  - 之后按加权序列填充（去重抽取，优先补空槽）。
3. **年末未处理事件机制**
  - 有害事件在年末按槽位顺序结算惩罚。
  - `地方总督坐大`是特殊升级调度事件：本年未处理不立即罚值，而是给该槽挂“次年升级”标记。
4. **槽位升级机制（Power Vacuum → Major Crisis）**
  - 下一个事件阶段开始时，若槽位仍是`地方总督坐大`且已挂标记，自动替换为`王权危机`。
5. **脚本历史战事：遗产战争（1667–1669）**
  - 可选“进攻”（脚本攻击）：花 2 经费，权力 +1，且有 50% 概率财政 +1。
  - 一旦进攻，进入“反法同盟压力”：自进攻当回合起 **5 年内**，每年以 40% 概率令抽牌尝试 -1（最终仍至少 1 抽）。
6. **第一关通胀触发机制**
  - 当 `财政 + 权力 + 合法性 >= 12` 时激活通胀。
  - 激活后，仅带 `通胀` 标签卡牌在“弃牌回洗入牌库”时按实例叠加费用。
7. **有限耐久牌耗尽后果**
  - `王室征收`/`王室干预`有“剩余次数”机制；耗尽后会从循环中移除并触发惩罚：
    - 征收耗尽：财政 -1
    - 干预耗尽：权力 -1

### 二、事件列表（费用 / 标签 / 处理与不处理作用 / 历史背景 / 发生机制）

> 说明：标签含义为 `有害/机遇/史实/持续`。`史实`仅叙事标识，不自带额外数值效果。

#### 1) 常规随机池（第一关）

1. **宫廷超支 `budgetStrain`**
  - 费用：2（支付经费处理）
  - 标签：有害
  - 处理：支付 2 经费解决
  - 不处理：财政 -1
  - 历史背景：凡尔赛扩建与宫廷礼仪推升常年支出
  - 发生机制：第一关常规池，权重 3
2. **巴黎骚动 `publicUnrest`**
  - 费用：仅可用干预处理（不能用经费直付）
  - 标签：有害
  - 处理：用`王室干预`（或功能等效牌）清除
  - 不处理：合法性 -1
  - 历史背景：粮价与流言引发首都骚动
  - 发生机制：第一关常规池，权重 3
3. **官僚拖延 `administrativeDelay`**
  - 费用：1
  - 标签：有害
  - 处理：支付 1 经费
  - 不处理：下回合抽牌修正 -1
  - 历史背景：地方官延缓执行中央政令
  - 发生机制：第一关常规池，权重 2；第 1 年固定开场前缀之一
4. **殖民贸易机遇 `tradeOpportunity`**
  - 费用：1
  - 标签：机遇
  - 处理：支付 1，经费结算后财政 +1
  - 不处理：无惩罚
  - 历史背景：海外航线与商人集团寻求王室支持
  - 发生机制：第一关常规池，权重 2；第 1 年固定开场前缀之一
5. **贵族阻挠 `politicalGridlock`**
  - 费用：2
  - 标签：有害
  - 处理：支付 2 经费
  - 不处理：获得`权力流失`3 回合（每回合抽牌尝试 -1）
  - 历史背景：大贵族联合牵制新税与政令推进
  - 发生机制：第一关常规池，权重 2
6. **地方总督坐大 `powerVacuum`**
  - 费用：2 或干预
  - 标签：有害 + 持续
  - 处理：支付 2 或用干预
  - 不处理：年末登记“升级为王权危机”（非立即数值惩罚）
  - 历史背景：地方权力中心绕开宫廷形成事实自治
  - 发生机制：第一关常规池，权重 1；属于升级调度源头
7. **王权危机 `majorCrisis`**
  - 费用：仅干预
  - 标签：有害 + 持续
  - 处理：必须干预
  - 不处理：每年末合法性 -1 且下回合抽牌修正 -1，并持续留场
  - 历史背景：中央权威受挑战后演变为结构性治理危机
  - 发生机制：仅由`地方总督坐大`未处理后在下一事件阶段转化产生

#### 2) 脚本历史事件（第一关）

1. **遗产战争 `warOfDevolution`**
  - 费用：脚本“进攻”消耗 2（非普通付费解事件）
  - 标签：史实
  - 处理：可发动进攻（权力 +1；50% 概率财政 +1）
  - 不处理：无直接惩罚，但会占位到 1669 窗口结束
  - 历史背景：1667–1668 年法兰西围绕西属尼德兰/弗朗什孔泰发动战争，促发欧洲制衡
  - 发生机制：仅在 1667–1669 年脚本注入

## 第 2 关：烈日长影（`secondMandate`，1676–1700）

- 回合：25 年（每回合 1 年）
- 开局资源：
  - 关卡定义值：财政 3 / 权力 3 / 合法性 3
  - 实际直开（standalone 草案默认）：财政 7 / 权力 7 / 合法性 5
  - 承接模式（continuity）：继承第一关结算资源（并按分支追加修正）
- 胜利条件（以运行时代码为准）：
  - 当前年需 ≥ 1696
  - `欧洲警觉`必须已解除（通常通过处理`里斯维克和约`）
  - 玩家身上不能挂有`遏制胡格诺-剩余势力`（`huguenotContainment`）状态；只要镇压链未完成（即`剩余势力`未归零），即便其他条件满足也不会胜利
  - 注：`src/data/levels.ts` 中第二关 `winTargets`（财政 10 / 权力 8 / 合法性 10）**仅作参考阈值，运行时不参与判胜**——`evaluateVictory` 对第二关只看上述三条。

### 一、机制列表（按“发生顺序/优先级”）

1. **欧洲警觉（开章+年度）**
  - 开章即时权力损失：当前版本为 **0**（已取消开章扣权机制）。
  - 年度基础漂移（回合开始）：
    - 设 `x = 财政 + 权力 + 合法性`，`y = 欧洲警觉进度`，`k = x - 12 - y*3`。
    - 若 `k > 0`：有 `k*10%` 概率进度 `+1`。
    - 若 `k < 0`：有 `(-k)*10%` 概率进度 `-1`。
    - 概率按 100% 封顶，进度始终夹在 `1~10`。
  - 年度外压注入（事件阶段）：
    - 进度 `1~5`：以 `进度*20%` 概率额外注入 1 个欧洲警觉事件。
    - 进度 `6~10`：保底注入 1 个，再以 `(进度-5)*20%` 概率注入第 2 个。
  - 阶段文本（状态栏背景）：
    - `1~2` 缓和，`3~4` 警觉，`5~6` 遏制，`7~8` 敌视，`9~10` 冲突。
2. **脚本历史线注入窗口**
  - 按年份窗口注入：`扩张余震/谨慎王权` → `奈梅亨和约` → `撤销南特敕令` → `奥格斯堡同盟` → `九年战争` → `里斯维克和约`。
  - 注：`地方战事`不属于脚本注入，而是欧洲警觉补充池中的额外外压事件。
  - **直开第二关（standalone）的第 1 年特例**：A 槽固定 `凡尔赛开支 versaillesExpenditure`，B 槽固定 `税收反弹 taxResistance`；这一回合跳过常规的程序事件填充，因此开场固定只有这 2 个事件。承接（continuity）模式下不触发此特例。
3. **反法情绪阈值机制**
  - 当 `权力 + 财政 > 20` 时激活`反法情绪`状态。
  - 当总和刚超过 20 时即立刻 `+1` 费用；之后每再满额外 `+5`，再 `+1`（即 `ceil((权力+财政-20)/5)`）。
  - 当该状态存在时，每回合结束向抽牌堆加入 1 张`反法遏制`（抽到时以 50%/50% 概率使权力 -1 或合法性 -1；打出费用为 `floor(欧洲警觉进度/2)`；打出后直接消失，不进入弃牌堆）。
  - 当降回阈值内时自动移除该状态并取消上述费用加成。
4. **宗教宽容的副作用触发**
  - 存在`宗教宽容`时，每回合事件相位末进行一次掷骰，按互斥三段 15% / 15% / 15%（合计 45%）至多注入三类宗派事件之一：`詹森派争议` / `阿米尼乌斯派争论` / `胡格诺地下网络骚动`；若三者已有任意一个占用槽位，或当前没有空槽，则跳过本次注入。
5. **胡格诺镇压链机制**
  - 选择“严厉镇压”后：添加`遏制胡格诺-剩余势力(3)`并向牌库加入 3 张`镇压胡格诺派`。
  - 每打出 1 张，剩余势力 -1；归零时移除该状态并清除所有该临时牌。
  - **胡格诺残余卷土重来**：只要`遏制胡格诺-剩余势力`仍在场，每**两次** `runEventPhase`（即正常情况下每两个回合一次，触发点在事件相位末）会向抽牌堆插入 1 张`镇压胡格诺派`，同时使`遏制胡格诺-剩余势力`+1，模拟地下网络的反复重建；状态被清零时该机制随之失效。
  - 注：`镇压胡格诺派` 在代码中实际标签为 `extra`（不是 `temp`）；其“打出后不入弃牌堆、不进入弃牌循环”的行为来自 `gameReducer.PLAY_CARD` 内的专用分支，而非 `temp` 标签。
  - **严格不变量**：`遏制胡格诺-剩余势力`的数字始终等于玩家整个牌库（抽牌堆 + 手牌 + 弃牌堆）中`镇压胡格诺派`卡牌的实例总数。每次新增/打出该卡或读档（`HYDRATE`/`normalizeGameState`）时都会即时通过 `enforceHuguenotContainmentInvariant` 同步：若卡数下降至 0，状态被移除并重置“卷土重来”计数器；若数字与实卡数不一致（旧存档/外部 HYDRATE），状态会被强制改写为实际卡数；游离在 `cardsById` 中但已不在任何区域的实例会被一并清理。

> 设计目标：在不显著提升复杂度的前提下，引入“法国（波旁）vs 奥地利哈布斯堡”的双边竞争感；核心体验是在 20 回合内通过**王位争夺进度条**与**哈布斯堡敌对牌池**（玩家回合结束后对手出牌）拉锯“王位继承战政治结果”，同时延续国内事件槽经营。

- 回合：20 年（每回合 1 年），历法年 **1701–1720**
- 你方：法国（波旁主张）；对手：**「对手：哈布斯堡家族」固定持续事件（见「事件列表」），以敌对牌库手牌**在玩家回合结束后出牌施压；**实力**为每回合出牌费用上限（见该事件）。**事件面板上展示敌方当前手牌**：每条为 **类型 emoji + 中文简称 + 对手费用标记 + 极简效果**（范式与「三、卡牌列表 — 1）」各牌 **UI 行**一致；**不展示**抽牌堆 / 弃牌堆）。
- 开局资源（建议，双模式）：
  - **承接模式（第二关 → 第三关）**：继承第二关结算时的财政 / 权力 / 合法性与牌组实例（含通胀与耐久），**过章整编**后再开局；第三关本局的 `calendarStartYear` 取第二关胜利当刻的历法年（见下「王位继承危机」一节）。
  - **主菜单直开（standalone）**：财政 **12** / 权力 **8** / 合法性 **10**（建议默认，便于不依赖前章体验机制）
- 胜负（建议）：
  - **王位争夺进度条**（关卡态，建议 **-10 ～ +10**）：**起始值**由历法 **1701** 年脚本事件**「王位继承危机」**当回合选择写入（**+1** 或 **-1**，见「事件列表」），非默认 0；到达 **-10** 立即失败；到达 **+10** 立即胜利（波旁稳定取得西班牙王位主张优势）。
  - **区间结局**：`-9～-4` 哈布斯堡优势；`-3～3` 妥协；`4～9` 波旁优势（非即时阈值时结算）。
  - **资源硬失败（即时）**：**合法性 = 0** 或 **权力 = 0** 立即失败（优先于区间结局）。

### 与第二关「撤销南特敕令」分支的衔接（已实现）

- 第二关若选择 **宗教宽容**：第三关开局向玩家**抽牌堆**注入 **4** 张 `religiousTensionCard`（**宗教冲突**，与第二关事件链奖励所用模板相同）。
- 第二关若选择 **严厉镇压**：第三关开局向玩家**抽牌堆**注入 **4** 张 `jansenistReservation`（**良心保留**，费用 2，打出后从牌库循环移除）。历史背景：胡格诺派被压制后，天主教内部出现詹森派抗命。**规则**：手牌从左向右排列时，若某张牌**紧右侧**为「良心保留」，则**该牌**被附加 **「抗命」** 标签；**带抗命的牌不可打出**。「良心保留」本身可正常打出。
- **主菜单直开第三关**（未承接第二关）：按 **镇压** 分支处理，即注入 **4** 张 `jansenistReservation`。
- 第三关开局**不再**应用早期实现草案中的「宽容 = 合法性 -1 + 永久宗教宽容 / 镇压 = 遏制胡格诺 + 三张镇压胡格诺」等与第二关事件重复的叠层；上述效果仅体现在本节的牌库注入。

### 「王位继承危机」与历法 / 回合对齐（脚本事件，1701 年）

- **触发历法年**：**1701** 年。实现上可与第一、二关相同：在 `scriptedCalendarEvents` 中为该事件配置 `presenceStartYear: 1701`，由 `applyScriptedCalendarPhase` 在对应历法年开始时注入空槽（参见 `src/logic/scriptedCalendar.ts`）。
- **主菜单直开第三关**：建议第三关 `calendarStartYear = 1701`，则**第 1 回合**的 `currentCalendarYear` 即为 1701，**第一回合**发生「王位继承危机」；玩家**结算该事件后**，在本关流程内**追加 / 揭示**固定事件**「对手：哈布斯堡家族」**（展示方式建议：占用 A/B/C 之一，或独立「对手」栏位；实现与 UI 以落地为准）。
- **承接第二关过关进入**：第三关开局时的 `calendarStartYear` = 第二关胜利结算时的历法年：
  - `calendarStartYear₃ = calendarYearForTurn("secondMandate", calendarStartYear₂, turn₂)`
  - 其中第二关默认 `calendarStartYear₂ = 1676`；`turnLimitRule` 为 `calendarEnd` 且 `endYear = 1701` 时，由 `getTurnLimitForRun` 得 **25** 回合，最后可玩历法年为 **1700**（`1676 + 25 - 1`）。若玩家在**最后一回合**获胜，常见情况是第三关从 **1700** 年起步，则 **1701** 年对应第三关**第 2 回合**。
- **通用公式**（第三关内，已知本关 `calendarStartYear₃`）：使事件在历法年 `Y` 出现的回合为  
`**turn = Y - calendarStartYear₃ + 1`**（与 `src/logic/scriptedCalendar.test.ts` 中脚本注入回合的推导一致）。  
**示例**：若承接后 `calendarStartYear₃ = 1696`，则 1701 年为第 **6** 回合；若为 1700，则 1701 年为第 **2** 回合。

### 一、机制列表（按“发生顺序 / 优先级”，设计草案）

1. **年初资源与抽牌机制**

3. **脚本历史线（含 1701 开章事件）**
  - **1701**：**王位继承危机**（见「事件列表」）——标志西班牙哈布斯堡绝嗣后的全欧继承真空与波旁 / 哈布斯堡两线对撞进入 playable 主线；**该事件结算完成后连锁注入**固定持续事件**「对手：哈布斯堡家族」**。  
  - **1713 起**：**乌得勒支和约**（见「事件列表」）——作为**结束向**脚本事件注入常规事件槽；与常驻的**「对手：哈布斯堡家族」**及**哈布斯堡敌对牌池**并行存在，在**六回合谈判窗口**内由玩家决定是否结束王位继承战争，**窗口耗尽则自动结束**（见该事件条目）。  
  - 其余年份窗口事件按产品拆分注入（继承合法性 / 欧陆战场 / 和约秩序三组，见事件列表）。
4. **王位争夺进度条结算**
  - 类比第二关「欧洲警觉」的关卡态展示：每回合在日志中拆成「你方贡献 / 对手贡献 / 净变化」。  
  - 你方：打出王朝主张 / 外交 / 和约向卡牌、处理关键事件。  
  - **哈布斯堡牌库（与玩家牌库相互独立）**：分 **抽牌堆**、**手牌**、**弃牌堆** 三区；共 6 张实例（3 种各 2），详见「三、卡牌列表 — 1）」。  
  - **回合开始**（第三关每个游戏年；建议与「年初」或玩家行动前最早阶段对齐）：对手**固定摸 2 张**——从**抽牌堆**顶依次入手牌；**需摸牌而抽牌堆为空**时，先将**弃牌堆**洗混成为新抽牌堆再继续摸；**弃牌堆亦空**则本段摸牌终止（通常表示全部实例仍在手牌中）。**含对手首次登场的回合**：上场前洗好 6 张入抽牌堆、手牌与弃牌堆为空，**第一次**执行本摸牌后**手牌恰为 2**，即**起始手牌两张**；之后每个回合开始**同样再摸 2**（手牌可累积，未打出者保留）。  
  - **时序**：上述摸牌之后进行玩家回合；**玩家回合内行动结束后** → **对手阶段**：仅能从**当前手牌**中选牌打出；打出的牌**结算效果后入弃牌堆**（不入对手抽牌堆）。  
  - **预算**：本回合对手已打出牌的**对手费用之和 ≤ 当前「实力」**；**能打出几张**取决于手牌中可选组合。  
  - **选牌**：具体选取顺序与优先级由**轻量 AI / 规则表**驱动（实现待定）。未打出的手牌**保留至下一回合**（草案无手牌上限）。  
  - **事件 UI**：**「对手：哈布斯堡家族」所在事件卡 / 栏位上实时列出敌方当前手牌**，每条采用 **UI 行**范式：**类型 emoji**（标识这张敌对牌）＋**简称**＋**对手费用**（建议 Unicode **①②③** 或 **💎1 / 💎2**，与实现择一）＋**效果摘要**（**序**＝王位争夺进度，**权/财/法**＝权力 / 财政 / 合法性，可配 📉 等状态向 emoji，见牌表「UI 行」示例）；**回合开始摸 2 之后、玩家行动前**即刷新，**对手阶段打牌后**再刷新；**不展示**抽/弃牌堆。  
  - **效果**：每张敌对牌的效果见牌池条目，可作用于**王位进度条**、三维资源、抽牌 / 经费修正等。
5. **三段式时代节奏系数（1701–1720）**
  - **1701–1707**：继承与开战期，进度条与**对手牌池**波动允许偏大。  
  - **1708–1713**：消耗与谈判期，波动趋缓、拉锯感增强。  
  - **1714–1720**：秩序重建期，大起大落概率降低，偏向锁定区间结局。

### 三、卡牌列表（费用 / 标签 / 效果 / 张数 / 历史背景）

#### 1) 哈布斯堡敌对牌池（仅对手使用，草案）

> **牌库结构**：与玩家牌库独立；分 **抽牌堆**、**手牌**、**弃牌堆**。  
> **实例构成**：**3 种牌各 2 张**（共 **6** 张）；**首次构建牌库**时洗混全部 6 张入**抽牌堆**，手牌与弃牌堆为空。  
> **每回合开始时**（含首个回合）：**固定摸 2 张**入手牌；**首次摸牌后手牌为 2**（即**起始手牌两张**），之后每回合开始**同样摸 2**（摸牌顺序：顶牌依次入手；**需摸牌而抽牌堆为空**时，先将**弃牌堆**洗入抽牌堆再继续摸；**弃牌堆亦空**则本段无可摸之牌——通常即全部牌仍在手牌中）。  
> **打牌**：仅在**玩家回合结束后的对手阶段**，从**手牌**中择牌打出；结算后对玩家生效，**牌入弃牌堆**。对手阶段内**手牌打光**不触发额外摸牌，**下一回合开始**再按「摸 2」补手（草案）。**循环**：当抽牌堆需补牌而为空时**洗回弃牌堆**，与玩家侧「牌库打空再洗」一致。  
> **对手费用**：每张牌有**对手费用**；**本回合已打出牌之对手费用总和 ≤ 当前实力**。  
> **效果**均作用于**玩家侧**（三维资源、`successionTrack`、抽牌 / 经费修正等）。名称与 ID 为占位，实现时与玩家卡 ID 区分（如前缀 `habsburg_`）。  
> **UI 行（事件上手牌列表用，草案）**：每条建议 `**类型emoji` `简称` `费用标` · `效果摘要`**（中间可用 **·** / **—** 分隔）；**费用标**对应**对手费用**（非玩家经费）。**效果摘要**用极短字：**序−1**＝王位争夺进度 −1，**权−1**＝权力 −1，必要时加 **📉** 表示对玩家不利走向（实现可选）。同名多张可重复一行或 **×2**。

### 四、状态列表（效果 / 发生原因）

1. **王位争夺进度 `successionTrack`（占位 ID，关卡态）**
  - 效果：范围 **-10～+10**；触及 ±10 立即胜负；区间内决定结局档位。  
  - 发生原因：第三关启用该关卡态时，**起始值**由**「王位继承危机」**选择写入（**+1** / **-1**）；之后每回合由玩家行为、**哈布斯堡敌对牌池**出牌与关键事件共同增减。
2. **哈布斯堡对手阶段（运行态，可选独立状态或仅日志）**
  - 效果：**回合开始**对手固定摸 2（抽牌堆空则洗弃牌堆）；**玩家行动结束后**，依**当前实力**从**手牌**打出敌对牌，效果表现为进度条左移 / 资源扣减 / 短时抽牌或经费修正等（以牌池条目为准）；打出牌入**弃牌堆**。  
  - 发生原因：**「对手：哈布斯堡家族」**注入后每回合触发。
3. **哈布斯堡实力 `opponentStrength`（占位 ID，关卡态，可选）**
  - 效果：整数 **N**，作为对手每回合出牌**费用总和**上限；可在事件文案中与**「对手：哈布斯堡家族」**合并展示而不单独占状态栏。  
  - 发生原因：**「对手：哈布斯堡家族」**注入时写入 **N = 2**；特定脚本事件可上调 / 下调 **N**（草案）。
4. **时代阶段修正（1701–1707 / 1708–1713 / 1714–1720）**
  - 效果：影响进度条波动幅度、事件权重或**对手牌池选牌**方差（实现可为隐式规则而非 `playerStatuses` 条目）。  
  - 发生原因：历法年进入对应阶段自动切换。
5. **过章整编临时态**
  - 效果：允许删牌与选牌；完成后写入第三关初始 `GameState`。  
  - 发生原因：第二关胜利 → 第三关 `bootstrap` 流程。
6. **乌得勒支和约谈判窗口 `utrechtTreatyCountdown`（占位 ID，关卡态或事件内嵌字段）**
  - 效果：整数 **6 → 0**，表示**乌得勒支和约**事件尚余几回合可主动选择「结束战争」；**每回合**若未选结束则 **−1**；**到 0** 时触发**自动结束**王位继承战争（与事件条目一致）。  
  - 发生原因：**历法 1713** 年脚本注入**乌得勒支和约**时写入 **6**；随玩家抉择递减或随自动结束清零。

> **叙事参考（导语 / 事件文案基底）**：西班牙哈布斯堡末王卡洛斯二世无嗣，帝国遗产危及欧洲均势；波旁以安茹公爵腓力（费利佩五世）系血缘与遗嘱自辩，哈布斯堡以奥地利大公查理系延续自辩；战争升级为大同盟式多国冲突。1713《乌得勒支和约》等确立费利佩可留西班牙王位但法西不得合并，条约均势取代单一普世王朝帝国想象。路易十四留下的强国家机器与欧洲遏制记忆，在本关可作为“高能力 / 高外部反弹”的双刃叙事。

## 给你的简短设计建议（面向迭代）

- **先保证“可读性”**：**哈布斯堡事件面板上常驻敌方手牌「UI 行」**（emoji + 简称 + 费用标 + 效果摘要），让玩家不翻牌表也能读懂威胁；进度条日志再补“为什么这一回合往左/往右”（含对手**打出了哪几张**、费用如何占用**实力**，以及必要时**摸 2** / **弃牌洗回抽牌堆**）。  
- **先保证“可逆性”**：至少给玩家两类手段能逆转**哈布斯堡牌池**带来的进度压力（外交类、谈判类）。  
- **先保证“可收束性”**：1713 年后把大起大落概率降低，否则结局会显得随机。  
- **先保证“与前两关连续”**：保留过章删卡 + 选卡，维持“王朝治理连续体”的手感。
