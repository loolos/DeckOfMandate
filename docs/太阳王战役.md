# 太阳王战役（代码对齐版）

> 本文档以当前实现为准：`src/data/*`、`src/app/*`、`src/logic/*`。

## 1. 章节与胜利条件

### 第一章 `firstMandate`（1661–1675）

- 回合上限：15（每回合 1 年）
- 起始：财政 2 / 权力 2 / 合法性 2
- 胜利：财政 >= 6，权力 >= 6，合法性 >= 5

### 第二章 `secondMandate`（1676–1700）

- 回合上限：25（每回合 1 年）
- 起始：财政 3 / 权力 3 / 合法性 3（但直开二章会通过开局草案覆写到更高资源）
- 数值目标：财政 >= 10，权力 >= 8，合法性 >= 10
- 额外门槛：
  - 当前年份 >= 1696
  - 已达成 `nymwegenSettlementAchieved`
  - 不能存在 `huguenotContainment`

## 2. 回合流程（当前引擎）

1. 年初状态资源结算（如赈济合法性+1）
2. 收入：经费 += 财政
3. 抽牌尝试计算：
   - 基础值由权力阈值函数决定（1,2,4,7,11,16...）
   - 再叠加：下一回合修正 + 排程修正 + 状态修正 + 反法同盟年度掷骰
   - 最终至少为 1
4. 抽牌（手牌上限 12；溢出弃置并记录日志）
5. 状态回合数递减（`religiousTolerance` / `huguenotContainment` 不自动递减）
6. 事件阶段：
   - 槽位升级转换（如地方总督坐大 -> 王权危机）
   - 清除已解决事件
   - 注入脚本事件
   - A-C 常规补位（基于确定性加权序列）
   - 额外注入（欧洲警觉、宗教矛盾、反法情绪扩散）
7. 行动阶段：打牌、支付经费处理、干预、脚本攻击、政策二选一
8. 年末：清空经费 -> 选保留手牌 -> 未保留弃置
9. 未解决有害事件按槽位顺序结算惩罚
10. 判定：合法性失败 / 胜利 / 超时失败 / 进入下一年

## 3. 第一章关键机制

- 起始牌组：`funding×4`、`crackdown×3`、`reform×2`、`ceremony×2`、`development×2`
- 开场事件前缀（第 1 回合）：`tradeOpportunity`、`administrativeDelay`
- `powerVacuum` 年末未处理会在下一年转成 `majorCrisis`
- `warOfDevolution`（1667–1669）可执行脚本进攻：
  - 花费 2 经费，权力 +1
  - 50% 概率财政 +1
  - 触发若干年“反法同盟抽牌风险”（每年 40% 概率 -1 抽牌尝试）
- 第一章通胀激活条件：`财政 + 权力 + 合法性 >= 12`

## 4. 第二章关键机制

- 常规随机池 10 种事件（不含 `frontierGarrisons`、`tradeDisruption`）
- 脚本线：`expansionRemembered/cautiousCrown`、`nymwegenSettlement`、`revocationNantes`、`leagueOfAugsburg`、`nineYearsWar`、`grainReliefCrisis`、`ryswickPeace`
- 欧洲警觉（`europeAlert`）存在时：
  - 开章立刻按 `floor(power/2)` 扣权力
  - 每年 50% 概率额外注入 `frontierGarrisons` 或 `tradeDisruption`
- 反法情绪扩散（`antiFrenchSentiment`）：
  - 当 `power + treasuryStat > 20` 激活
  - 每超过阈值 5 点再增加额外常规事件注入数量
- 南特撤销事件（`revocationNantes`）：
  - 选择“宗教宽容”会获得长期状态，并每年有 30% 概率额外触发 `religiousTension`
  - 选择“严厉镇压”会获得 `huguenotContainment` 并引入 `suppressHuguenots` 临时牌链

## 5. 事件处理口径

- `crackdown` 与 `diplomaticIntervention` 仅能选择“有害且未解决”事件。
- `nymwegenSettlement` 处理成功后会设置章节目标标记，并产生权力 -2、合法性 -2 的代价。
- 非持续危机类事件在年末惩罚后通常清除；持续类（continued）会保留等待后续处理。
- 主线脚本历史事件（如遗产战争、奈梅亨和约、撤销南特敕令等）使用「史实」标签，而不是「机遇」标签；「史实」仅用于说明“历史真实发生”，本标签本身不提供额外机制效果。

## 6. 卡牌通胀与耐久

- 带 `inflation` 标签卡牌按“实例”累计通胀费用。
- `funding` / `crackdown` / `diplomaticIntervention` / `development` 为耐久牌（有限使用次数）。
- 耐久耗尽后会触发额外后果：
  - `funding` 耗尽：财政 -1
  - `crackdown` 耗尽：权力 -1
  - `diplomaticIntervention` 耗尽：仅日志提示

## 7. 二章开局模式

### 直开二章（standalone）

- 视为已走第一章军事分支：`warOfDevolutionAttacked = true`，`europeAlert = true`
- 使用“旧牌快照 + 二章固定新增包”构建牌库
- `reform/ceremony` 默认带初始通胀层（由草案写入）

### 承接二章（continuity）

- 继承第一章真实牌实例（含通胀与剩余耐久）
- 只能删牌，最多删 3 张
- 再追加二章固定新增包：`grainRelief×2`、`taxRebalance×2`、`diplomaticCongress×1`
